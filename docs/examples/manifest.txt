# Manifest of cmake-xray docs/examples/ files (AP M5-1.8 A.2).
#
# Each non-comment, non-blank line names exactly one example file in this
# directory followed by its SHA-256 hash, separated by whitespace (the
# `sha256sum` output convention). tests/validate_doc_examples.py is the
# CTest-registered drift gate (`doc_examples_validation`):
#   1. asserts manifest entries match directory contents (no extra files,
#      no missing entries),
#   2. recomputes the SHA-256 of each example and verifies it matches the
#      manifest hash byte-for-byte,
#   3. runs the per-format validator (JSON via tests/validate_json_schema.py
#      against spec/report-json.schema.json, DOT via
#      tests/validate_dot_reports.py, HTML via tests/validate_html_reports.py)
#      where applicable.
#
# Regenerate the hashes when an example is intentionally refreshed:
#
#   ( cd docs/examples && sha256sum * | sort ) > docs/examples/manifest.txt.new
#
# The trailing `| sort` keeps the manifest in stable hash-prefix order so
# diffs across regens stay tight; without it `sha256sum *` follows the
# directory traversal order and the gate diff churns on every run.
